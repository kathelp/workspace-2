# Support Category Taxonomy

Start simple. Expand only when the inbox demands it.

## Initial Categories

- account access
- billing
- bug report
- feature request
- setup / onboarding
- how-to question
- integration issue
- performance / outage
- docs confusion
- permissions / roles
- data issue
- other

## Product Area

Track the product area separately from the issue type.

Example product-area fields:
- authentication
- dashboard
- reporting
- integrations
- billing
- user management
- API
- data import / export

## Rules

- Prefer one primary category.
- Add secondary categories only if truly useful.
- Avoid category explosion early.
- Change taxonomy only when repeated patterns justify it.
