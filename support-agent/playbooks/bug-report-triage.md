# Bug Report Triage

## Goal
Determine whether the issue is a bug, user confusion, setup problem, or external dependency.

## Gather
- user goal
- expected behavior
- actual behavior
- exact error text
- steps to reproduce
- account / workspace / environment context
- when it started
- whether anyone else is affected

## Fast Triage

### Likely user confusion
- missing setup step
- permission issue
- unsupported workflow
- stale expectation from old docs

### Likely bug
- reproducible unexpected behavior
- regression
- conflicting system state
- multiple users affected

### Likely escalation
- data loss
- security concern
- billing impact
- production outage

## Response Shape
1. Brief acknowledgement
2. One or two clarifying questions or a concrete next step
3. Escalation note if needed
